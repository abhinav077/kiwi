# Kiwi — Design System Direction

## Reference direction

The approved visual direction combines:

- Editorial typography and expressive hero headings.
- Layered pastel cards and category-coded schedules.
- Dense but readable timelines.
- Floating dock-like navigation.
- Organic blobs, soft paper texture, and restrained botanical elements.
- Calm, nature-inspired warmth.

The references are inspiration, not assets to copy. Kiwi must have its own identity.

### Usage rule

Use ivory and cream for breathing space. Use deeper tones for readable text, selected states, progress, and primary actions. Use pale tints for surfaces. Do not use every accent on every screen.

## Typography

- Primary UI font: a clean, readable rounded sans-serif available on Android.
- Display font: an editorial serif or expressive high-contrast face only if licensing and Android availability are confirmed.
- Diary accent: a readable handwritten font used only for journal text and personal accents.
- Never use decorative fonts for health data, task details, dates, or error messages.

## Shape and depth

- Cards: 20–32dp radius depending on importance.
- Pills/chips: fully rounded.
- Primary actions: strong contrast, generous height, clear label.
- Shadows: soft and restrained; prefer tonal separation and layered fills over heavy elevation.
- Organic backgrounds: use subtle, low-opacity shapes behind content; never reduce text contrast.

## Final design quality bar

Every screen should have a memorable focal point, a clear primary action, readable information density, meaningful color rhythm, and a calm empty state. The app should not look like a generic Material template with a pastel overlay.
