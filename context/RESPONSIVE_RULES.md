# Kiwi — Responsive Rules

## Width classes

- Compact phone: below 360dp — one column, reduce decorative field size, preserve 20dp padding.
- Phone: 360–599dp — default layouts, bottom dock.
- Tablet: 600dp and above — centered max-width 720dp; use two columns only where it improves scanning.

## Rules

- Never stretch cards edge-to-edge on large tablets.
- Keep primary actions within thumb reach on phones.
- Preserve the same color semantics and hierarchy across widths.
- Prefer scrolling over shrinking text below the token sizes.
- Support portrait first; landscape may use a wider centered canvas.
