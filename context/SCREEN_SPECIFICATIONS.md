# Kiwi — Screen Specifications

## Welcome

- Base: `KiwiPaper` with oversized organic coral/sage/lavender fields.
- Top: small Kiwi wordmark and optional minimal botanical mark.
- Center: editorial greeting and one warm supporting sentence.
- Lower third: illustration/abstract composition, never a stock dashboard screenshot.
- Bottom: full-width forest primary action, 56dp high, 28dp radius.

## Today

- Header: avatar/initial, greeting, notification icon, overflow.
- Hero: “Make today feel good, [name]” with a progress ring or expressive completion figure.
- Task summary: compact segmented status row and a visible completion count.
- Task list: grouped by Morning/Afternoon/Evening; category tint and time are visible at a glance.
- Self-care: one small actionable card, not a second dashboard.
- Bottom dock: Today, Calendar, Diary, Ask Kiwi, More. Selected item uses dark pill/high contrast.

## Calendar

- Header: back/title/date actions.
- Date rail: seven circular date cells; selected date is forest or charcoal with light text.
- Timeline: time labels on the left, cards aligned to a vertical guide, current-time marker.
- Timeline card anatomy: colored leading rail, title, time, optional participants/avatar, status chip.
- Floating add action is visible without obscuring the timeline.

## Add task

- Focused top bar with close and confirm actions.
- Fields are grouped into title, description, date/time, priority, project/tags.
- Use large pill inputs and a sticky forest create button.
- Validation is inline and calm; never use red-only error communication.

## Wellness and Diary

- Wellness: record-first cards, date context, historical chart/summary, clear caution copy where relevant.
- Diary: cream writing surface, generous line height, optional peach paper accent, mood row, private lock affordance.

## Tablet adaptation

- Keep the same visual language, but cap content width at 720dp and use a centered composition.
- Calendar may use two columns: date/timeline and day summary.
- Today may use a 2-column hero/task arrangement when width allows.
